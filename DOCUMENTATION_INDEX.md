## 🔄 Keeping Docs Updated

When making code changes:

1. **New API endpoint?**
   - Update [API.md](API.md)
   - Update types in [AGENTS.md](AGENTS.md)

2. **New component?**
   - Document in [AGENTS.md](AGENTS.md) (component section)
   - Add feature description to [FEATURES.md](FEATURES.md) if user-facing

3. **New feature (user-facing)?**
   - Add to [FEATURES.md](FEATURES.md)
   - Update [CHANGELOG.md](CHANGELOG.md)
   - Consider updating [README.md](README.md)

4. **Bug fix?**
   - Add to [CHANGELOG.md](CHANGELOG.md) under "Fixed"

5. **Architectural change?**
   - Update [ARCHITECTURE.md](ARCHITECTURE.md)
   - Update [AGENTS.md](AGENTS.md) if component changes
